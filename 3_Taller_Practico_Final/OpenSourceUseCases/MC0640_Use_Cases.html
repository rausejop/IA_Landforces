§ 1 · Executive Summary

What this report contains and how to read it

This Technical Report presents 32 Use Cases for the Land Tactical CIS profile PFL-00403, drawn from the Information Exchange Requirement tables of Annex C to Draft MC 0640. Each UC is generated under the NAF v4 + ADatP-47 methodology and grounded in AJP-5 (Operational Planning) and AJP-6 (CIS) doctrine.

Every UC card carries two parts: a NAF v4 · ADatP-47 Specification Header (formal IER ID, primary actor, goal, trigger, pre/postconditions, NAF view, FMN service, releasability, doctrinal basis) and a 14-row Operational Body (the six IER dimensions plus equipment, time window, environment, zones, availability, classification, risk meter, and attack surface). The "Copy" button on each card emits a Word-ready HTML table — paste directly into a Word document.

Scope

Brigade · Battle Group · Company. MC 0640 covers LANDCOM downward; this report focuses on Bde-and-below. Upper echelons (Division, Corps, LANDCOM) appear only as upstream reach-back context.

Method

NAF v4 NOV/NSOV views + ADatP-47 SysML-style UC packages. AJP-5 supplies commander's intent; AJP-6 mandates the service-oriented stance. STEADFAST COBALT supplies the technical constraints.

Realisation

FMN Spiral 5 / NISP Baseline 16 services on a typical NATO stack: Systematic SitaWare on Klas Voyager, routed by Cisco ESR6300, into Thales Synaps / L3Harris AN/PRC-163 SDRs, with Advenica / Fox-IT CDS at every NNI.

2.3 — Why "deterministic"

Determinism here means three things:

  • Schema-fixed. Every UC has the exact same 16-field NAF header + 14-row body (see §8 Use Case Framework). No structural variation between UCs.
  • Rule-derived. Field values are produced by explicit derivation rules from the canonical inputs. The IER ID, FMN service, NAF view, releasability, and C3 Taxonomy L1/L2 are computed; only operational specifics (goal, trigger, pre/postconditions, attack-surface narrative) are hand-curated and per-UC traceable.
  • Reference-traceable. Every technical assertion either cites a Reference (A–R in §11) or names a STANAG / AJP / RFC inline. The Annex B formal-definitions table is the dictionary against which every term in this report can be checked.

§ 4 · Introduction & Doctrinal Basis

What an IER is, and why every Use Case picks one

IER — Information Exchange Requirement

Per AJP-6, an IER is a category of information that must be exchanged between operational facilities so commanders can decide. MC 0640 makes each IER answer six dimensions: Who, What, When, Protection, Precedence, Volume. In this project we treat each row of Annex C as one Serial (S-NNN) paired 1:1 with one Use Case (UC-NNN).

14-row Use Case card

Each UC card renders 14 fields in fixed order — six IER dimensions, then equipment, time window, environment, zones/CoI, availability, classification, an LPI/LPD risk meter with a one-sentence narrative, and an attack-surface note. Mission classification overrides the IER's notional default.

Use Case generation standard — NAF v4 + ADatP-47

In NATO, Use Cases are not authored ad-hoc. They are generated under the NATO Architecture Framework (NAF) v4 and detailed procedurally by ADatP-47 (Allied Data Publication 47). AJPs supply the doctrinal context (AJP-5 for operational planning, AJP-6 for CIS), not the UC template. Every card in §05 carries a NAF v4 · ADatP-47 spec header above the 14-row operational body.

LayerStandardRole for UC generation
Mother frameworkNAF v4Defines NOV (Operational View) + NSOV (Service-Oriented View). UCs are the bridge from operational need to technical solution.
Procedural data std.ADatP-47Working-group rules for interoperability design. Operational UCs drafted in SysML / UML-adapted military methodology.
Op. planning doctrineAJP-5Commander's intent + lines of operation. Engineers convert these into technical UCs.
CIS doctrineAJP-6Mandates service-oriented, UC-validated systems. Prohibits unrooted system development.

Anti-pattern: citing an AJP as the UC origin. The UC origin is the NAF view + ADatP-47 model; AJPs justify it.

Both are NATO Military Committee directives, but the scopes do not overlap. This deliverable visualises only MC 0640 (Brigade and below). LANDCOM, Corps and Division are shown only as upstream reach-back nodes — their primary UC content belongs to MC 0593.

§ 5 · Technical Constraints — STEADFAST COBALT lessons learned

Four constraint blocks that shape every Use Case

1. Day-Zero Protected Core Networking

National units down to Brigade level must not route raw IP across boundaries. All cross-NNI data rides STANAG 4711 IPsec PCN tunnels over an agnostic black transport backbone. Every Brigade spins up a PCR mapped into the FMN Mission Network Element; NNI dynamic routing uses BGP-4.

STANAG 4711PCRBGP-4NNIBlack transport

2. Echelon-tiered bandwidth & protocol

Corps/Div HQ: SATCOM + fibre, IPv4/IPv6 dual-stack, MIP4 baseline, full HD video. Bde/Bn: MarLOS STANAG 4691 + SATURN Ed.4 PRF-151 FFH for air-ground. Coy/Plt: XMPP for chat, AComP-5634 MELPe 2.4 kbps VARC half-duplex PTT. Standard VoIP is blocked at the edge.

STANAG 4691 MarLOSSATURN Ed.4STANAG 5651 ESSORSTANAG 5634 NBWFAComP-5634 MELPeXMPP RFC 6120

3. NNI data integrity — STANAG 4774 labels + CDS

"Weakest link" doctrine: a single tactical-CP compromise at a small nation can traverse the federated core. Inter-nation boundaries enforce Cross-Domain Solutions with STANAG 4774 confidentiality metadata labels. NFFI (STANAG 5527) BFT updates from lower-classification networks must pass label validation before ingress to NATO Secret COPs.

STANAG 4774STANAG 5527 NFFICDSXML metadata label

4. SWaP-C — NFV / DCIS at the edge

Migration off heavy single-purpose racks to Network Functions Virtualization on Deployable CIS nodes. Coy-to-Bn Command Posts run on a single COTS ruggedized SFF blade hosting routing engine, XMPP server, local C2 registry, and security gateway. Stand-up target: under 30 minutes.

§ 8 · Use Case Framework

Every UC has the same structure — here is the field-by-field rationale

All 32 Use Cases in §9 share two fixed blocks: a NAF v4 / ADatP-47 Specification Header (16 fields, formal metadata per ADatP-47) and a 14-row Operational Body (the six IER dimensions per AJP-6 Reference K, plus eight operational context fields). This chapter documents the rationale, range of values, and meaning of each field so any UC card in §9 can be read against a single framework definition.

B.1 — Operational & doctrinal terms

TermStandard where definedFormal definition
ACOAllied Command OperationsAAP-06 · NATO Command StructureOne of NATO's two Strategic Commands, responsible for the planning, preparation, and execution of all Alliance military operations. Headquartered at SHAPE (Supreme Headquarters Allied Powers Europe), Mons, Belgium. Its sister command is Allied Command Transformation (ACT).
AJPAllied Joint PublicationAAP-06The principal NATO joint-doctrine publications, organised in series (AJP-1 to AJP-6+, with dependent sub-publications). Each AJP defines authoritative doctrine for a specific operational domain or function and is formally ratified by NATO nations.
AORArea of ResponsibilityAAP-06The geographical area associated with a specific commander within which that commander has the authority and the responsibility for the conduct of military operations.
BFTBlue Force TrackingAAP-06 · STANAG 5527 (NFFI)A capability that displays the location of friendly (blue) forces in near real time on a Common Operational Picture. Technical realisation in NATO uses the NATO Friendly Force Information (NFFI) protocol per STANAG 5527.
BGBattle GroupAJP-3.2A combined-arms tactical formation built around a reinforced battalion-sized manoeuvre unit, employed as the principal manoeuvre element for tactical land operations. Typical AOR is of the order of 4 × 5 km.
BJFC / BGIFC / BFDCBrigade / Battlegroup Joint Fires Cell · Brigade Fire Direction CentreAJP-3.9 · MC 0640 Annex CThe brigade-level cell that plans, coordinates and executes joint fires (BJFC); the battlegroup-level equivalent (BGIFC); and the brigade computational cell that converts a call-for-fire into firing data for the gun batteries (BFDC). Doctrine: AJP-3.9 (Allied Joint Doctrine for Joint Targeting); the specific entity-to-entity flows are tabulated in MC 0640 Annex C.
CCIRCommander's Critical Information RequirementsAJP-2 · AJP-5The comprehensive list of information needs identified by the commander as critical to facilitating timely decision-making. Comprises Priority Intelligence Requirements (PIR), Friendly Force Information Requirements (FFIR), and Essential Elements of Friendly Information (EEFI).
CFSTCompany-Level Fire Support TeamAJP-3.9 · national fire-support doctrineA small fire-support element attached at company level, responsible for processing calls-for-fire from forward elements and coordinating with the battlegroup joint fires cell.
CIFC / LIFCCombined / Land Intelligence Fusion CellAJP-2Multinational (Combined) or land-specific (Land) intelligence fusion cells that aggregate, analyse and fuse intelligence from multiple sources, producing unified intelligence products for distribution to subordinate commanders.
COISTCompany Intelligence Support TeamAJP-2 · national tactical-intelligence doctrineA small intelligence cell organic to a manoeuvre company, responsible for processing tactical-level intelligence (patrols, sensors, HUMINT contacts) and feeding the battlegroup intelligence section.
CoICommunity of InterestNISP Baseline 16 · FMN FrameworkA collaborative group of users sharing a common interest in, or operational need for, an exchange of information. In FMN, a CoI defines the releasability scope of a body of information and the set of services that produce and consume it.
COPCommon Operational PictureAJP-6 · AAP-06A single identical display of relevant information shared by more than one command, facilitating collaborative planning and assisting all echelons to achieve situational awareness.
EW / EWCCElectronic Warfare / EW Coordination CellAJP-3.6 · AAP-06EW: military action employing electromagnetic and directed-energy actions to control the electromagnetic spectrum or to attack the adversary. EWCC: the staff cell that plans and coordinates EW activities at a given command level.
HUMINTHuman IntelligenceAJP-2.1A category of intelligence derived from information collected and provided by human sources, including overt collection, interrogation, and source operations.
IERInformation Exchange RequirementAJP-6 (Reference K)Per AJP-6: "those categories of information that must be exchanged between operational facilities to provide commanders with essential information for decision-making." MC 0640 requires each IER to answer six dimensions: Who, What, When, Protection, Precedence, Volume.
ISTARIntelligence, Surveillance, Target Acquisition, ReconnaissanceAJP-2 · AAP-06An integrated intelligence and operations function that fuses information collected by surveillance, target-acquisition and reconnaissance sensors and processes it into intelligence products for the commander.
LANDCOMAllied Land CommandNATO Command StructureNATO's command for all Allied land forces, headquartered in Izmir, Turkey. In MC 0640, LANDCOM is the upper boundary of tactical CIS scope; this Technical Report treats LANDCOM as upstream reach-back context only — its primary content belongs to MC 0593.
LOGSITREPLogistics Situation ReportAPP-11 · AJP-4A standard NATO MTF report on the logistics state of a unit: personnel, POL (petroleum, oil, lubricants), ammunition, equipment, casualties, maintenance.
MTFMessage Text FormatAPP-11A formal, machine-readable structured-text format used for NATO operational messages (e.g., SITREP, ASSESSREP, 9-Line CASEVAC). Each MTF has a published schema and a defined field set; APP-11 is the NATO Message Text Formatting System publication.
QRFQuick Reaction ForceAAP-06 · AJP-3.2A pre-designated force held at high readiness to respond rapidly to unforeseen tactical situations within a unit's AOR.
RFIRequest For InformationAAP-06 · AJP-2A formal mechanism by which a unit requests additional information or intelligence from a higher, lateral, or supporting headquarters or agency.
SITREPSituation ReportAPP-11A standard NATO MTF report providing a periodic update on the tactical situation of a reporting unit.
TETTarget Engagement TeamAJP-3.9 · national targeting doctrineA small cell that converts an approved target into a specific engagement using available fires assets; often co-located with the brigade fires cell.
VJTF(L)Very High Readiness Joint Task Force (Land)NATO Readiness Action Plan · Wales Summit 2014The land component of NATO's Very High Readiness Joint Task Force — a brigade-sized formation able to deploy within days. Established under the NATO Readiness Action Plan agreed at the Wales Summit (2014); held on a rotational basis by NATO nations.

B.2 — CIS, networking & waveform terms

TermStandard where definedFormal definition
BGP-4Border Gateway Protocol v4IETF RFC 4271The standard inter-domain routing protocol of the Internet, used in NATO Protected Core Networking to announce subnet reachability between national Mission Network Elements. NATO FMN profiles add TCP Authentication Option (TCP-AO, RFC 5925) and RPKI-style origin validation.
C3Consultation, Command & ControlAJP-6 · AAP-06 · NATO C3 BoardThe activities and supporting systems that enable Alliance political consultation and military command and control. The NATO C3 Taxonomy (managed by the NC3B) organises NATO services into hierarchical Level 1 / Level 2 categories used throughout MC 0640 Annex C.
CDSCross-Domain SolutionNCIA · NISP B16 · STANAG 4774A controlled interface that enforces release policy between security domains of different classifications, typically using a label-aware filter and an audit chain. In NATO FMN, the CDS reads STANAG 4774 Confidentiality Metadata Labels at the NNI to authorise or downgrade information transfer.
DCISDeployable CISNCIA doctrineNATO term for a deployable CIS package — a transportable command-post-grade CIS stack including networking, services, and security functions, often virtualised. Underpins the STEADFAST COBALT push to virtualised tactical clouds.
DSCPDifferentiated Services Code PointIETF RFC 2474 / 2475A 6-bit field in the IPv4 Type-of-Service / IPv6 Traffic-Class header that marks per-packet QoS treatment. Standard codes include EF (Expedited Forwarding) for voice and the AF (Assured Forwarding) classes for differentiated data.
EPMElectronic Protection MeasuresAJP-3.6The division of EW concerned with measures taken to protect friendly use of the electromagnetic spectrum from the effects of friendly or hostile EW actions.
ESSOR HDREuropean Secure Software-defined Radio · High Data Rate WaveformSTANAG 5651 · OCCAR ESSOR ProgrammeBroadband mobile ad-hoc network waveform developed under the European ESSOR consortium (managed via OCCAR), supporting IP-native operation on SDR platforms primarily at brigade and battalion vehicular level.
FFHFast Frequency HoppingSTANAG 4372 (SATURN) · AJP-3.6Electronic-protection technique in which the transmitter and receiver pseudo-randomly change carrier frequency many times per second following a shared hop pattern. Baseline for SATURN and integral to NBWF / ESSOR HDR EPM modes.
FMNFederated Mission NetworkingNISP B16 · AC/322 FMN FrameworkAn initiative that enables rapidly formed mission networks among NATO nations and partners using a common set of standards, processes, and services. Currently at Spiral 5, codified by NISP Baseline 16 / AC/322-D(2024)0053.
IGMPv3Internet Group Management Protocol v3IETF RFC 3376The IPv4 host-router protocol for managing multicast group membership. Used in tactical IP networks to distribute COP overlays, RAP feeds and NFFI BFT streams.
LPI / LPDLow Probability of Intercept / Low Probability of DetectionAAP-06 · AJP-3.6LPI: transmission characteristics designed so that an adversary signal-intelligence receiver has a low probability of intercepting the emission. LPD: low probability that the receiver detects an emission is present at all. Typically achieved through spread-spectrum, frequency hopping, and low-duty-cycle techniques.
MarLOSMulti-channel Ad-hoc Relay · Line-Of-SightSTANAG 4691NATO standard for a multi-channel tactical line-of-sight relay capable of operating in an ad-hoc mesh topology, for medium-bandwidth packetised IP at brigade and battalion level.
MELPeMixed Excitation Linear Prediction enhancedSTANAG 4591 (codec) · STANAG 5634 / AComP-5634 (NBWF profile)A low-rate (typically 600, 1200, or 2400 bit/s) speech codec adopted by NATO for narrowband secure-voice channels. MELPe-2400 is the default for NBWF voice. Codec definition: STANAG 4591; NBWF integration profile: STANAG 5634 / AComP-5634.
MIP4Multilateral Interoperability Programme · Baseline 4MIP Programme docsThe fourth baseline of the MIP land C2 data model, providing a common information model for land operations that allies' systems can publish and consume.
MNEMission Network ElementFMN Framework DocumentsA national or coalition CIS contribution to a Mission Network. Each MNE federates into the Mission Network through standardised NNIs.
NAFNATO Architecture FrameworkNAF v4 · NATO C3 Board · ADatP-47NATO's authoritative architecture framework for C3 systems. NAF v4 mandates the use of Operational (NOV) and Service-Oriented (NSOV) views, with Use Cases bridging operational need to technical solution. ADatP-47 codifies the SysML/UML artefact set.
NBWFNATO Narrowband WaveformSTANAG 5630 · STANAG 5634 / AComP-5634NATO's narrowband tactical waveform for VHF/UHF SDR operation in 25 kHz / 50 kHz channels, optimised for contested EW environments with MLPP-prioritised IP transport. Physical layer: STANAG 5630; data + voice profiles: STANAG 5634 / AComP-5634.
NFFINATO Friendly Force InformationSTANAG 5527NATO standard XML/JSON schema for exchanging friendly-force position and status data — the protocol underlying tactical Blue Force Tracking.
NFVNetwork Functions VirtualizationETSI NFV ISG · NATO DCIS adaptationArchitectural approach in which network functions (router, firewall, XMPP server, etc.) are realised as software workloads on virtualised commodity hardware rather than dedicated appliances. Baseline definitions in ETSI NFV ISG documents; NATO DCIS guidance adapts them for tactical use.
NISPNATO Interoperability Standards and ProfilesAC/322-D(2024)0053 (Baseline 16)The NATO catalogue of standards and service profiles (PFLs) mandatory or recommended for NATO and FMN interoperability. Baseline 16 (current at issue) is AC/322-D(2024)0053.
NNINetwork-to-Network InterfaceITU-T · NISP profile usageThe boundary point where two distinct networks (typically of two different administrative authorities) interconnect. In NATO it is the place where STANAG 4774 label policy and CDS rules are enforced.
PCN / PCRProtected Core Networking · Protected Core RouterSTANAG 4711PCN: NATO architecture for a black-core transport service that carries encrypted user data across an agnostic shared physical layer between coloured-side enclaves, implemented via IPsec tunnels. PCR: the PCN-aware edge router that establishes those tunnels. Each Mission Network Element typically spins up at least one PCR.
PFLProfile (NISP)NISP Baseline 16Within NISP, a PFL-XXXXX is a published interoperability profile that constrains the choice of standards and parameters for a specific service. PFL-00403 is the Land Tactical C2 Information Exchange Profile (FMN Spiral 5) — the target profile of this Technical Report.
PTTPush-To-TalkSTANAG 5634 / AComP-5634A half-duplex voice control mode in which the operator must press and hold a button to transmit; the universal control paradigm of tactical radio voice. STANAG 5634 / AComP-5634 defines the VARC half-duplex signalling for MELPe over NBWF.
RTPReal-time Transport ProtocolIETF RFC 3550End-to-end transport protocol for real-time data such as audio and video. Used to carry MELPe voice packets over UDP/IP in NATO tactical IP networks.
SATURNSecond-generation Anti-jam Tactical UHF Radio for NATOSTANAG 4372NATO standard for anti-jam UHF tactical voice, the successor to Have Quick II. Edition 4 / PRF-151 is the variant referenced by PFL-00403 for air-ground voice.
SDRSoftware Defined RadioNATO SDR Initiative · industry-standard termA radio in which signal-processing functions traditionally implemented in hardware (modulation, filtering, demodulation) are realised in reprogrammable software, allowing a single chassis to run multiple waveforms — typically ESSOR HDR, NBWF, SATURN, and legacy analogue VHF.
SFFSmall Form Factorindustry · NATO DCIS guidanceCommercial classification for compact ruggedised computer enclosures usable in tactical vehicles or man-portable kits. Pairs with SWaP-C optimisation goals.
SWaP-CSize, Weight, Power, Costindustry · STEADFAST COBALT lessons learnedThe four critical optimisation axes for deployable military electronics. Lower SWaP-C means more mobility and a smaller electromagnetic / thermal signature; explicitly surfaced as a STEADFAST COBALT lesson learned.
UDPUser Datagram ProtocolIETF RFC 768Connectionless datagram transport on IP; bearer for RTP voice, NFFI BFT, XMPP server-to-server traffic, and many other tactical IP services.
XMPPExtensible Messaging and Presence ProtocolIETF RFC 6120 (core) · RFC 6121 (IM & presence)Open standard for near-real-time messaging, presence, and request-response interactions. Adopted by FMN as the tactical chat protocol of choice due to its low bandwidth footprint and extensibility. RFC 6120 defines the core protocol; RFC 6121 defines instant messaging and presence semantics.
